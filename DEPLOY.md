# 部署指南

如果您想让其他人通过公网访问您的网站，请按照以下步骤操作。

## 1. 为什么本地运行别人无法访问？
当您在本地电脑运行项目时（如 `localhost:5173`），这个地址仅在您的电脑上有效。即使您将电脑连接到 WiFi，也只能通过局域网 IP 让同一 WiFi 下的人访问，外网用户无法直接连接。

要实现公网访问，您有两种选择：
1. **正式部署（推荐）**：购买云服务器，将项目部署上去，获得一个公网 IP。
2. **临时演示**：使用内网穿透工具（如 cpolar, ngrok）将本地端口暴露给外网。

## 2. 正式部署步骤 (基于 Docker)

本项目已经为您准备好了 Docker 部署文件，可以一键在服务器上运行。

### 准备工作
1. **购买服务器**：阿里云、腾讯云等（建议安装 CentOS 7+ 或 Ubuntu 20.04+）。
2. **安装 Docker**：在服务器终端运行安装命令（参考 Docker 官方文档）。
3. **安装 Docker Compose**。

### 部署操作
1. **上传代码**：
   将整个项目文件夹上传到服务器（可以使用 Git 或 SCP）。
   ```bash
   git clone <您的仓库地址>
   cd tiaozhanbei
   ```

2. **启动服务**：
   在项目根目录下运行：
   ```bash
   docker-compose up -d --build
   ```
   *这将自动构建前端、后端镜像，并启动 MySQL 数据库。*

3. **访问**：
   打开浏览器，输入服务器的公网 IP 地址即可访问（默认端口 80）。

### 环境变量
如果需要配置阿里云 OSS 或其他密钥，请在启动前设置环境变量，或者修改 `docker-compose.yml` 中的 environment 部分。

## 3. 常见问题
- **端口冲突**：确保服务器的 80 和 8080 端口没有被占用。
- **防火墙**：请在云服务器的安全组设置中，放行 **TCP 80** (前端) 和 **TCP 8080** (后端) 端口。
